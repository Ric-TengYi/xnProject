# Task Plan: xnProject 3.18 最新需求落地

## Goal
基于 2026-03-19 最新 134 项需求清单，按业务优先级推进 3.18 版本落地，先完成 P0 主链路闭环，再逐步补齐主数据、报表、预警、安全、车辆/车队、小程序与外部对接能力。

## Current Baseline
- 最新需求状态表：`docs/tech-plans/requirements_status_2026-03-19.md`
- 当前代码状态：前后端可编译，合同结算、登录权限、组织角色日志已有较完整骨架

## Risks
- 合同相关后端正处于活跃修改中，需避免覆盖现有未提交改动
- 项目/车辆/场地部分后端 controller 仍为空或仅基础 CRUD，前端联调时可能需要补最小接口
- 小程序目录当前未见有效业务源码，后续需要确认技术选型与代码基线
- 多个原型页面使用静态数据，接入真实接口时要统一字段口径
- 车辆 patch 涉及 `biz_vehicle` 历史表结构，老库需继续维持手动 patch/回填流程

## Decisions Made
| Decision | Rationale |
|----------|-----------|
| 先做 P0 核心链路 | 直接覆盖 3.18 最关键业务价值 |
| 状态表单独落盘到 `docs/tech-plans` | 便于持续更新与追踪 |
| 当前阶段先优先前后端真实闭环，不先做大而全重构 | 降低风险，尽快形成可演示版本 |
