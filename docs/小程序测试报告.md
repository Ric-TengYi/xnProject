## 一、实现与联调概况

### 1.1 后端实现（xngl-service）

| 接口 | 说明 | 实现状态 |
|------|------|----------|
| POST /api/mini/auth/login | 小程序登录（username、password，tenantId 默认 1） | ✅ 已实现，复用 AuthService，返回 token、user.id/name/role |
| GET /api/mini/excavation-orgs/current | 当前单位（需 JWT） | ✅ 已实现，根据当前用户 mainOrgId 查 OrgService 返回 orgName、userId、userName |
| GET /api/mini/excavation-orgs/projects | 可用项目列表（需 JWT） | ✅ 已实现，当前返回模拟 2 条；可后续接入 biz_project |
| GET/POST /api/mini/photos | 出土拍照列表与上传 | ✅ 已实现（stub），列表与提交返回 200 + 模拟/占位数据 |
| GET/POST /api/mini/checkin-exceptions | 打卡异常申报 | ✅ 已实现（stub） |
| GET/POST /api/mini/delay-applies | 延期申报 | ✅ 已实现（stub） |
| GET/POST /api/mini/feedbacks、POST /api/mini/feedbacks/{id}/close | 问题反馈 | ✅ 已实现（stub） |
| GET /api/mini/drivers/home、tasks、alerts | 司机端聚合与列表 | ✅ 已实现（stub） |

- **JwtAuthFilter**：已放行 `/api/mini/auth/login`（及 path 包含 `/mini/auth/login` 的请求），无需 token 即可调用小程序登录。若本地 POST 该接口仍返回 401「未登录或 token 缺失」，请**重启后端**使配置生效。
- **CORS**：`/api/**` 已配置 `allowedOriginPatterns("*")`，H5 开发态可请求后端。

### 1.2 前端实现（xngl-mp）

- 登录页、首页（当前单位、项目、快捷入口）、出土拍照、打卡异常申报、延期申报、问题反馈、司机端页面均已实现，与上述接口一一对接。
- **请求封装**：`src/utils/request.ts` 统一 baseURL（开发态默认 `http://localhost:8090`）、token 存取、401 跳转登录。
- **开发态 baseURL**：已在 `config/dev.ts` 中配置 `process.env.TARO_APP_API_BASE` 为 `http://localhost:8090`，便于本地联调。

### 1.3 联调前置条件

1. **启动后端**：在 `xngl-service-starter` 目录执行 `mvn spring-boot:run`（需先停止占用 8090 的旧进程），确保加载 Mini* 控制器。
2. **启动 H5**：在 `xngl-mp` 目录执行 `npm run dev:h5`，浏览器访问提示的地址（默认 http://localhost:10086）。
3. **验证登录**：  
   - 小程序登录：`curl -X POST http://localhost:8090/api/mini/auth/login -H "Content-Type: application/json" -d '{"username":"admin","password":"admin"}'`  
   - 应返回 `code:200`、`data.token`、`data.user.id/name/role`。

## 三、H5 与浏览器实测（本次补充）

### 3.1 H5 编译与可访问性

- **编译**：在 `xngl-mp` 下执行 `npm run dev:h5`，编译成功（`Compiled successfully`），dev-server 监听 `http://localhost:10086`。
- **根路径只有 ~/ 或空白问题与修复**：开发态下 `dist` 常为空，webpack-dev-server 对根路径 `/` 会返回目录列表页（标题/内容里只有 `~ /`），导致“10086 端口还是空页面，只有~/字符”。修复方式：
  1. 在项目根目录新增 `index.template.html`，内容为 SPA 壳子，**按顺序**引入 `<script src="/js/runtime.js"></script>`、`<script src="/js/app.js"></script>`（缺 runtime 会白屏）；
  2. 在 `config/index.ts` 的 `h5.devServer` 中配置 `setupMiddlewares`，对请求 `/` 或 `/index.html` 直接返回该模板内容；
  3. 在 `config/dev.ts` 的 `h5.output` 中配置开发态使用固定文件名 `js/[name].js`，使 `/js/runtime.js`、`/js/app.js` 可访问。
- **本机 HTTP 验证**：修复后 `curl http://localhost:10086/` 返回 **200** 且为完整 HTML（含 `/js/app.js`），`curl http://localhost:10086/js/app.js` 返回 **200** 且为 bundle 内容。
- **结论**：H5 在本机已就绪，**请在本机浏览器**打开 `http://localhost:10086` 做端到端验证。

### 3.2 浏览器自动化（MCP）说明

- 使用 cursor-ide-browser 访问 `http://localhost:10086/` 与 `http://127.0.0.1:10086/` 时，页面表现为「listing directory /」，而非小程序登录页。
- 可能原因：MCP 浏览器运行环境与本机不同，其 localhost/127.0.0.1 可能指向其他服务（如工作区静态服务），因此**未在本环节完成登录→首页→快捷入口的自动化点击测试**。
- **建议**：以本机浏览器为准，手动执行下方「人工验证步骤」。

### 3.3 人工验证步骤（可落地）

1. **启动**  
   - 后端：`cd xngl-service/xngl-service-starter && mvn spring-boot:run`（确保 8090 为带 Mini 接口的进程）。  
   - H5：`cd xngl-mp && npm run dev:h5`。
2. **打开**  
   - 浏览器访问：`http://localhost:10086`。
3. **登录**  
   - 账号 `admin`，密码 `admin`，点击登录 → 应跳转首页。
4. **首页**  
   - 查看「当前单位」「可用项目」及 5 个快捷入口（出土拍照、打卡异常申报、延期申报、问题反馈、司机端）。
5. **快捷入口**  
   - 依次点击：出土拍照、打卡异常申报、延期申报、问题反馈、司机端 → 各页面能打开、无白屏/控制台报错即可视为通过。

完成上述步骤后，可将 13-小程序 用例中的「需重启后端联调」等说明改为「已在本机浏览器验证通过」。

### 4.2 分模块执行说明

- **登录（M-L）**：清除 token 后打开 10086 → 应跳转登录页；不填账号/密码点登录 → Toast 提示；admin/admin 登录 → 跳转首页（后端需放行 mini 登录并重启）。
- **首页（M-H）**：登录后查看当前单位、可用项目、点击 5 个快捷入口（出土拍照、打卡异常申报、延期申报、问题反馈、司机端）→ 各进入对应页面。
- **出土拍照 / 打卡异常 / 延期 / 反馈（M-P/M-C/M-D/M-F）**：进入页面 → 列表加载（可为空）→ 点击新增 → 弹窗必填校验 → 提交后刷新列表。
- **司机端（M-V）**：进入司机页 → 工作台与任务/预警展示；点击快捷入口跳转对应功能。
- **接口（M-API）**：浏览器 F12 Network 查看请求头是否带 `Authorization: Bearer <token>`；清除 token 后请求需登录接口 → 401 并跳转登录页。
