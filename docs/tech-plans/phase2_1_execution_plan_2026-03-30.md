# Phase 2.1 执行方案：经营基础与主数据运行链路整改验收

## 任务概览

**目标**: 完成单位管理、项目清单、处置证、打卡、消纳、场地、车辆等主数据模块的 UI 收口、权限回归、业务闭环验收。

**现状**:
- ✅ 后端权限体系已完成（Phase 1）
- ✅ 11/11 业务回归通过（phase71_business_master_regression）
- ⏳ 需补齐：UI 缺陷、权限穿透、多租户数据隔离、自动化回归脚本

**完成标准**:
1. UI 缺陷关闭（列表越界、筛选拥挤、详情混乱、按钮状态）
2. 权限回归通过（平台管理员、租户管理员、普通角色、字段限制）
3. 业务闭环验收（查询、新增、编辑、导出、消息跳转）
4. 自动化证据落盘（测试报告写入 docs/test-reports）
5. 验收结论明确（可交付/条件通过/阻塞待修）

## 技术执行分解

### 1. 前端整改轨道（Cursor-Frontend）

**范围**: 7 个主数据模块的 UI 收口

| 模块 | 页面 | 优先级 | 关键问题 |
|------|------|--------|---------|
| 单位管理 | 列表/详情 | P0 | 列表越界、筛选栏拥挤、按钮状态 |
| 项目清单 | 列表/详情/交款 | P0 | 详情区块混乱、交款流程交互 |

**交付物**:
- 每个模块的 UI 缺陷清单（含截图）
- 修复后的组件代码

### 2. 后端整改轨道（Codex-Backend）

**范围**: 权限校验、数据口径、接口联调

#### 2.1 权限穿透检查

**验证点**:
- [ ] 单位查询：租户用户只能看自己租户的单位
- [ ] 项目查询：用户只能看权限范围内的项目
- [ ] 处置证查询：按组织隔离
- [ ] 场地查询：按组织隔离
- [ ] 车辆查询：按组织隔离

**实现**:
- 在 `MenusController`、`PermissionsController`、`RolesController` 中集成 `TenantManagementAccessGuard`
- 为每个主数据查询接口添加权限过滤
- 验证多租户、多组织下的数据隔离

#### 2.2 数据口径修正

**检查项**:
- [ ] 字段返回完整性（是否缺少必要字段）
- [ ] 字段权限限制（某些字段是否应该隐藏）
- [ ] 关联数据一致性（项目-场地-处置证-车辆-消纳记录的关联）
- [ ] 导出数据准确性

**交付物**:
- 权限过滤实现代码
- 数据口径修正清单
- 接口联调验证报告

### 3. 自动化测试轨道（Cursor-Test）

**范围**: 模块级回归脚本、权限穿透测试、业务闭环验证

#### 3.1 复用已有脚本

- `phase71_business_master_regression` 中的 11 个用例作为基础
- 补充权限穿透测试用例
- 补充多租户隔离测试用例

#### 3.2 新增专项脚本

**脚本清单**:
1. `test_master_data_permission_filter.js` - 权限过滤验证
2. `test_master_data_multi_tenant_isolation.js` - 多租户隔离
3. `test_master_data_ui_regression.js` - UI 缺陷回归
4. `test_master_data_business_flow.js` - 业务闭环验证

**验证场景**:
- 平台管理员：全部数据可见
- 租户管理员：本租户数据可见
- 普通角色：权限范围内数据可见

**交付物**:
- 自动化测试脚本
- 测试报告（写入 `docs/test-reports/phase2_1_master_data_regression_2026-03-30.md`）

### 4. 验收轨道（Claude-Acceptor）

**范围**: 综合验收、结论输出、遗留问题清单

#### 4.1 验收标准

| 维度 | 标准 | 判定 |
|------|------|------|
| UI 完整性 | 所有缺陷关闭 | ✅/❌ |
| 权限正确性 | 权限回归 100% 通过 | ✅/❌ |
| 业务闭环 | 关键流程可用 | ✅/❌ |
| 自动化覆盖 | 脚本覆盖率 ≥80% | ✅/❌ |
| 性能指标 | 查询 <500ms，列表渲染 <1s | ✅/❌ |

#### 4.2 验收结论

输出三种结论之一：
1. **可交付** - 所有标准通过，可进入生产
2. **条件通过** - 部分非关键问题遗留，可交付但需后续跟进
3. **阻塞待修** - 关键问题未解决，不可交付

#### 4.3 交付物

- 最终验收报告
- 遗留问题清单（优先级、影响范围、修复计划）
- 后续整改建议
