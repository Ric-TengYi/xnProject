# 移动端架构选型 - 微信 / 钉钉 / H5 公用

**目标**：移动端功能（出土拍照、打卡异常、延期申报、问题反馈、司机任务等）需在**微信小程序、钉钉、以及 H5** 中公用，创建前先确定技术路线。

---

## 一、方案对比

| 维度 | 原生小程序（各端一套） | H5 单页应用 | 跨端框架（Taro / uni-app） |
|------|------------------------|-------------|----------------------------|
| **形态** | 微信原生 WXML/WXSS、钉钉原生 DSL，各写一套 | 一套 React/Vue 打 H5 包，用 URL 打开 | 一套 React/Vue 编译出多端产物 |
| **微信** | 微信小程序原生 | 公众号/企微内 H5、或小程序 web-view 嵌 H5 | **微信小程序** 原生形态 |
| **钉钉** | 钉钉小程序原生 | 钉钉工作台添加「H5 应用」填 URL | **钉钉小程序** 原生形态 |
| **其他** | - | 浏览器、企微、飞书等只要可打开 URL 即可 | 可同时出 H5、支付宝/抖音等小程序 |
| **代码复用** | 两套以上，几乎无复用 | **一套**，与现有 Web 技术栈一致可复用 | **一套**，多端共用业务逻辑与组件 |
| **体验** | 各端原生，体验最好 | 依赖 WebView，性能与动效略逊 | 小程序端接近原生，H5 同 H5 方案 |
| **与现有 xngl-web 一致** | 否（不同语法） | 是（可同用 React + 同一 API） | 是（Taro 用 React，与 xngl-web 一致） |
| **发布与审核** | 各端分别提审、发版 | H5 发版即生效，无商店审核 | 小程序需各端提审，H5 同 H5 |
| **维护成本** | 高（多套代码） | 低（一套） | 低（一套，多端编译） |

---

## 二、推荐结论：**跨端框架（Taro + React）**

- **一套代码** 编译出：**微信小程序**、**钉钉小程序**、**H5**，满足「微信、钉钉都可公用」且保留 H5 兜底。
- 技术栈与现有 **xngl-web（React）** 一致，组件与请求层可抽象复用，后端继续沿用 `技术方案_小程序.md` 中的 `/api/mini/*` 接口。
- 小程序端使用**原生小程序形态**，体验与审核符合平台要求；钉钉/企微等也可通过「工作台 H5」配置同一 H5 链接作为补充。

**次选**：若近期只做**验证与快速上线**、暂不上架小程序，可先做 **H5 单页**（React，与 xngl-web 同栈），后续再迁到 Taro 或与 Taro 并存（H5 入口共用后端）。

---

## 三、技术选型细节（Taro + React）

- **框架**：Taro 3.x/4.x（React），与现有 Web 一致。
- **构建**：`npm run build:weapp` / `build:dd` / `build:h5` 分别出微信、钉钉、H5。
- **路由**：Taro 内置路由，按小程序页面配置；H5 使用同一路由表。
- **请求**：封装 `request`，baseURL 指向现有后端（与 xngl-web 同域或同网关），token 与 `/api/mini/auth` 登录方式对齐。
- **UI**：Taro UI 或 NutUI（React）等，保证小程序与 H5 组件一致。
- **差异**：扫码、定位、推送等通过 `Taro.xxx` 或 `process.env.TARO_ENV` 做多端适配；仅小程序或仅 H5 的逻辑用条件编译或分支。

---

## 四、与现有后端、文档的衔接

- **身份与接口**：沿用 `技术方案_小程序.md`：小程序复用现有用户体系（`user_id`）、统一附件与 `/api/mini/*` 分组。
- **功能范围**：登录绑定、出土拍照、打卡异常申报、延期申报、问题反馈、司机任务等，接口按该文档实现；移动端只做对应页面与调用。
- **现有 manager 包**：`MiniProgramUserController` 等可保留或逐步迁移到 web 层 `/api/mini/*`，与 Taro 前端统一对接。

---

## 五、目录与工程（已创建）

- 已在仓库内新增 **xngl-mp**（与 `xngl-web` 平级）：
  - **技术栈**：Taro 3.6 + React 18 + TypeScript。
  - **产出**：微信小程序（`npm run build:weapp`）、钉钉小程序（`npm run build:dd`）、H5（`npm run build:h5`）。
  - **入口**：`src/app.tsx`、`src/app.config.ts`；页面：`src/pages/index`（首页）、`src/pages/login`（登录占位）。
  - **配置**：`config/index.ts`（含 H5、钉钉插件）；微信/钉钉 AppID 在 `project.config.json` / `project.dd.json` 中替换。
- 使用说明见 **xngl-mp/README.md**。

**报告结束**。移动端工程已按本选型创建，可在此基础上对接 `/api/mini/*` 与 `技术方案_小程序.md` 中的功能。
