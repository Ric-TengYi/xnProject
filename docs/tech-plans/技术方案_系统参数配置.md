---

## 3. 分功能技术方案

### 3.1 列表和编辑

#### 1. 数据库设计

- 新增 `sys_config_param`，字段建议为 `param_code/param_name/param_group/param_type/param_value/default_value/secret_flag/description/status`。
- 新增 `sys_config_param_history` 记录变更前后值、操作人、变更原因。

#### 2. 接口定义

- `GET /api/system-params`：分页查询参数。
- `PUT /api/system-params/{id}`：编辑参数。
- `GET /api/system-params/{id}/history`：查看历史。

#### 3. 业务逻辑

- 参数建议按 `BUSINESS/ALERT/INTEGRATION/SECURITY` 分组，支持字符串、数字、布尔和 JSON 四类值。
- 敏感参数只允许密文展示，历史记录中也不能回显完整明文。

#### 4. 前端实现要点

- 可先在系统设置下新增一个参数配置页，也可用 `Placeholder.tsx` 先承接。
- 前端表格按分组、关键字和启停状态筛选，编辑时显示数据类型和默认值。

## 4. 跨功能公共约束

### 4.1 参数边界

- 系统参数只保存运行时配置，不保存大体量业务数据。
- 敏感参数统一走脱敏展示，严禁日志打印明文值。

## 5. 验收标准

- 支持系统参数列表、编辑和历史查看。
- 敏感参数具备脱敏展示和审计能力。
