# 端到端测试计划 - 平台与司机端联调

**目标**：与平台结合测试，平台创建账号后在本端登录；司机端账号由平台「默认开启司机端」创建，小程序登录后仅具备司机工作台能力。

**环境**：后端 8090（MySQL + Redis），Web 前端 5173，小程序开发工具或真机（API 指向 8090）。

---

## 一、前置数据准备

1. **执行数据库 patch**：确保已执行 `009_driver_role_seed.sql`，使 demo 租户下存在角色「司机」(role_code=DRIVER)。
2. **种子账号**：`006_local_admin_seed.sql` 已提供 admin/admin（TENANT_ADMIN），用于平台端登录。

---

## 二、测试流程概览

| 阶段 | 平台侧（Web 管理端） | 被测端 | 验证点 |
|------|----------------------|--------|--------|
| 1. 登录 | - | Web 登录页 | 使用平台已有账号 admin 登录，进入首页 |
| 2. 创建普通用户 | 组织与人员 → 新增人员，用户类型选「平台用户」，分配角色 | - | 该账号可在 Web 登录，具备管理端功能 |
| 3. 创建司机端账号 | 组织与人员 → 新增人员，用户类型选「司机端（仅小程序司机工作台）」，所属组织、角色（含「司机」） | - | 该账号仅用于小程序，登录后仅见司机工作台 |
| 4. Web 登录 | - | Web /login | 用新创建的「平台用户」账号登录，能进入首页/各模块 |
| 5. 小程序-平台用户 | - | 小程序登录 | 用「平台用户」账号登录，进入首页：当前单位、可用项目、快捷入口（出土拍照、打卡异常、延期申报、问题反馈、司机端） |
| 6. 小程序-司机端 | - | 小程序登录 | 用「司机端」账号登录，**直接进入司机工作台**，无「当前单位/可用项目/出土拍照」等，仅有：今日任务、预警、处置证、打卡异常申报、问题反馈 |

## 三、分模块测试顺序（与平台结合）

### 模块 1：登录（平台 + Web + 小程序）

- **1.1 平台账号登录 Web**
  - 步骤：打开 Web `/login`，账号 admin，密码 admin。
  - 预期：POST /api/auth/login 200，跳转 `/`，localStorage 有 token、userInfo（含 userType/tenantId）。
- **1.2 平台创建「平台用户」**
  - 步骤：Web 登录后进入「组织与人员」`/settings/organization` → 新增人员：姓名、账号、用户类型「平台用户」、所属组织、角色（如 TENANT_ADMIN 或其他），初始密码。
  - 预期：POST /api/users 200，列表刷新出现新用户。
- **1.3 平台创建「司机端」账号**
  - 步骤：新增人员，用户类型选「**司机端（仅小程序司机工作台）**」，所属组织、角色至少包含「司机」（系统自动带出司机角色）。
  - 预期：POST /api/users 带 userType=DRIVER、roleIds 含司机角色；列表可查该用户。
- **1.4 Web 登录 - 新平台用户**
  - 步骤：退出后用 1.2 创建的账号登录 Web。
  - 预期：登录成功，进入首页，可访问各管理端菜单（依角色权限）。
- **1.5 小程序登录 - 平台用户**
  - 步骤：小程序登录页输入 1.2 创建的账号密码，调用 POST /api/mini/auth/login。
  - 预期：返回 200，data.user.role 非 DRIVER，data.user.userType 非 DRIVER；跳转至「首页」（当前单位、可用项目、快捷入口）。
- **1.6 小程序登录 - 司机端账号**
  - 步骤：小程序登录页输入 1.3 创建的司机端账号密码。
  - 预期：返回 200，data.user.role=DRIVER 或 data.user.userType=DRIVER；**跳转至「司机工作台」**，不经过首页；再次打开小程序（已登录）也应直接进入司机工作台。

### 模块 2～14

按《模块功能测试执行清单》顺序执行：场地、车辆、合同、用户/组织、角色、数据字典、审批、日志、预警、单位、项目、处置证、小程序业务（打卡异常、延期、反馈等）。  
每模块：平台侧如需先建数据则在 Web 完成，再在对应端（Web 或小程序）执行用例并记录实际结果。

## 四、司机端能力边界（验收标准）

- 司机端账号在小程序登录后：
  - **仅能进入**：司机工作台（今日任务、预警、处置证、打卡异常申报、问题反馈）。
  - **不能进入**：首页「当前单位 / 可用项目 / 出土拍照 / 延期申报」等平台侧入口（通过进入首页即重定向到司机工作台实现）。
- 平台侧：组织与人员中「用户类型」为「司机端」时，保存时后端应收到 userType=DRIVER，且角色包含司机；小程序登录接口返回 role/userType 供前端跳转。
