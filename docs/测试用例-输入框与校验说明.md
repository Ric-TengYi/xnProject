# 测试用例 - 输入框与校验限制说明

本文档对《测试用例.xlsx》各模块的测试细节补充至**输入框级别**的限制与校验要求，便于执行时精确验证。

## 1. 登录模块

| 输入项 | 类型 | 必填 | 限制/校验 | 异常用例要点 |
|--------|------|------|-----------|--------------|
| 账号 | Input | 是 | 无 maxLength，非空校验 | 空提交应提示「请输入账号」 |
| 密码 | Password | 是 | 非空校验 | 空提交应提示「请输入密码」 |
| 手机号(验证码Tab) | Input | 是 | 正则 `^1[3-9]\d{9}$`，11位 | 非11位/非1开头应提示「请输入正确手机号」 |

## 测试执行说明

1. **正向用例**：按上表使用合法长度、格式、范围内数据提交，预期通过并持久化或返回正确结果。
2. **异常用例**：  
   - 必填项：留空提交，预期前端 rules 提示或接口 4xx 及明确错误信息。  
   - 格式：如手机号非 1[3-9] 开头的 11 位、InputNumber 超出 min/max，预期前端校验或接口校验报错。  
   - 分页：pageNo/pageSize 越界或非法，预期不 500，返回空列表或统一错误结构。
3. **CRUD**：  
   - 查：列表筛选、分页、详情 ID 有效/无效。  
   - 增：必填项完整/缺失、格式合法/非法。  
   - 改：同增，且 ID 存在/不存在。  
   - 删：存在 ID 删除、不存在 ID 或已删记录，预期明确提示或 4xx。

与《测试用例.xlsx》中用例编号（TC001–TC118）对应执行时，可将本说明作为「测试数据准备」与「预期结果」中输入框级别的细化依据。
