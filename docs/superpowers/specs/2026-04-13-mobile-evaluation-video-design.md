# 小程序正式评审视频设计

## 目标

基于现有系统能力，输出一支不高于 15 分钟、以动态操作为主的小程序正式评审视频。视频需优先覆盖评分项中的移动端功能、PC 看板与统计报表，同时补充个人中心、账号绑定、问题反馈等亮点能力，提升完整性展示效果。

## 范围

### 一、移动端核心评分能力

使用 `xngl-web/src/pages/MiniProgramDemo.tsx` 作为主要移动端演示载体，动态展示以下流程：

1. 小程序登录
2. 出土打卡
3. 消纳清单查看
4. 手动消纳
5. 事件上报
6. 延期申报

### 二、PC 端评分能力

使用现有 Web 页面动态展示以下能力：

1. 场地容量看板
2. 项目进度看板
3. 地图车辆分布与轨迹回放
4. 总体分析
5. 运力分析
6. 场地报表
7. 项目报表

### 三、增值亮点能力

在评分项之后补充以下内容，用于完整性加分：

1. 移动端当前身份与组织信息
2. 账号绑定与 OpenID 免登
3. 问题反馈
4. 历史留痕回看
5. 车辆监管回看

## 设计原则

1. 评分项优先，非评分亮点后置。
2. 全程采用真实页面动态录屏，不使用静态截图拼接替代核心操作。
3. 移动端提交后的结果，要在移动端留痕区或 PC 端看板/报表中体现同步回写。
4. 讲解结构按“移动端作业 -> 平台联动 -> 统计展示 -> 增值亮点”展开，而不是按系统菜单机械逐页翻。

- `SitesReports.tsx`：场地报表
- `ProjectsReports.tsx`：项目报表

## 录屏结构

### 第一段：移动端核心评分流程

1. 登录移动端
2. 出土打卡提交
3. 消纳清单查看
4. 手动消纳提交
5. 事件上报提交
6. 延期申报提交

### 第二段：平台联动看板

1. 场地容量看板
2. 项目进度看板
3. 地图分布与轨迹回放

### 第三段：统计分析报表

1. 总体分析
2. 运力分析
3. 场地报表
4. 项目报表

### 第四段：亮点补充

1. 当前身份与绑定状态
2. 问题反馈
3. 历史留痕回看
4. 车辆监管补充说明

## 风险与处理

1. 若部分数据实时刷新速度不稳定，则通过“提交后刷新页面/切换页签”的方式展示同步结果。
2. 若个别页面按钮受真实数据影响不可点，录屏脚本应提供容错分支，至少保证页面级能力展示不丢失。
3. 若移动端历史数据不足，应优先通过录屏前自动提交样例数据补齐展示数据。
