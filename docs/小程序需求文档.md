# 小程序需求文档（xngl-mp 实现范围）

**依据**：`docs/tech-plans/技术方案_小程序.md`、`消纳功能描述.xlsx` 中的小程序模块。  
**目标**：明确仅由小程序端（微信/钉钉/H5）实现的功能点、用户故事与验收标准，便于单独排期与实现。

---

## 二、分功能需求与验收标准

### 2.1 登录与绑定（P0）

**用户故事**：作为现场人员，我使用小程序登录/绑定已有账号，以便在移动端使用出土、打卡、申报等功能。

**功能点**：
- 支持微信/钉钉/H5 的登录入口（code 或 token 换取 session）。
- 若未绑定：展示「绑定账号」引导（手机号+验证码或账号密码），绑定后与现有 `user_id` 关联。
- 若已绑定：进入出土单位首页；支持退出/切换账号（可选）。

**验收标准**：
- [ ] 未登录时打开小程序，可进入登录/绑定页。
- [ ] 绑定成功后，可进入首页且后续请求携带有效 token。
- [ ] 未绑定时有明确文案引导，无空白或报错。

**接口依赖**：`POST /api/mini/auth/login`（入参：code 或 phone+code / username+password；出参：token、user 简要信息）。

---

### 2.2 出土单位首页（P0）

**用户故事**：作为已绑定用户，我在首页看到当前单位、可操作项目及待办事项，以便快速进入对应功能。

**功能点**：
- 首屏展示：当前单位名称、可用项目列表（可点击进入项目维度操作或筛选）。
- 待办事项：如待拍照、待申报、待处理预警等条数或列表入口（具体与后端聚合接口一致）。
- 快捷入口：出土拍照、打卡异常申报、延期申报、问题反馈、司机端（若为司机角色）等。

**验收标准**：
- [ ] 登录后进入首页，展示当前单位与项目列表（可为空）。
- [ ] 点击项目可进入项目相关操作或筛选（如出土拍照按项目筛选）。
- [ ] 快捷入口可跳转至对应功能页。

**接口依赖**：`GET /api/mini/excavation-orgs/current`、`GET /api/mini/excavation-orgs/projects`（或聚合首页接口）。

---

### 2.3 出土拍照（P1）

**用户故事**：作为现场人员，我能在现场拍照上传并填写备注，并查看自己的拍照记录。

**功能点**：
- 拍照：调起相机拍照，可选项目/场地，必填或选填备注；上传时带拍摄时间、定位（若授权）。
- 列表：我的拍照记录，按时间倒序，支持按项目筛选；展示缩略图、时间、项目、审核状态。

**验收标准**：
- [ ] 可调起相机拍照并选择项目（下拉或选择器）。
- [ ] 上传成功后列表可看到新记录；列表支持下拉刷新与按项目筛选。
- [ ] 未授权定位时仍可上传，仅不传经纬度。

**接口依赖**：`POST /api/mini/photos`（body：project_id、site_id、photo_type、file、remark 等）、`GET /api/mini/photos`（query：project_id、pageNo、pageSize）。

---

### 2.4 打卡异常申报（P1）

**用户故事**：当打卡记录有误或需申诉时，我能在小程序中选择该记录、填写原因并上传佐证，并查看申报进度。

**功能点**：
- 申报表单：选择原打卡记录（列表或搜索）、异常类型、原因说明、佐证图片上传。
- 我的申报：列表展示申报时间、关联打卡、类型、状态（待审核/通过/驳回/已关闭）；支持查看详情。

**验收标准**：
- [ ] 可选择一条打卡记录并填写原因、上传图片后提交。
- [ ] 提交成功后可在「我的申报」中看到记录及状态。
- [ ] 列表支持分页或下拉加载。

**接口依赖**：`POST /api/mini/checkin-exceptions`、`GET /api/mini/checkin-exceptions`。

---

### 2.5 延期申报（P1）

**用户故事**：我需要对合同或项目申请延期时，在小程序提交延期申请并查看历史。

**功能点**：
- 申报表单：业务类型（如合同/项目）、关联业务 ID 或选择项目、申请延期至日期、原因、附件。
- 历史列表：展示申请时间、关联项目/合同、申请延期至、状态（待审/通过/驳回）。

**验收标准**：
- [ ] 可选择项目（或业务类型+业务 ID）、填写延期截止日期与原因后提交。
- [ ] 提交成功后可在列表中看到该条申请及状态。
- [ ] 若后端返回材料缺失提示，前端展示明确文案，不盲提。

**接口依赖**：`POST /api/mini/delay-applies`、`GET /api/mini/delay-applies`。

---

### 2.6 问题反馈（P2）

**用户故事**：我在现场发现问题时，能通过小程序提交文字、图片或位置，并查看处理进度。

**功能点**：
- 提交：反馈类型、内容描述、图片上传、可选位置；关联项目/场地可选。
- 列表：我的反馈列表，状态为未处理/处理中/已关闭；可查看详情与处理说明。

**验收标准**：
- [ ] 可提交文字与图片，可选填位置与项目/场地。
- [ ] 列表展示状态与处理进度，详情可查看处理说明（若后端返回）。

**接口依赖**：`POST /api/mini/feedbacks`、`GET /api/mini/feedbacks`、`POST /api/mini/feedbacks/{id}/close`（若支持用户关闭）。

---

### 2.7 司机专属（P1）

**用户故事**：作为司机，我打开小程序后能看到当日任务、允许路线、处置证、预警与打卡入口，并快速进入异常申报与问题反馈。

**功能点**：
- 司机首页：五块信息——任务、路线、证件、预警、打卡（可为卡片或列表摘要）。
- 任务：当日任务列表，含项目、场地、状态等。
- 预警：待处理预警条数或列表入口。
- 快捷入口：异常申报、问题反馈。

**验收标准**：
- [ ] 司机角色登录后可见司机首页（或首页增加司机 Tab/入口）。
- [ ] 首页展示任务、路线、证件、预警、打卡等入口或摘要。
- [ ] 点击可进入对应列表或详情；异常申报与问题反馈与通用入口一致。

**接口依赖**：`GET /api/mini/drivers/home`（聚合）、`GET /api/mini/drivers/tasks`、`GET /api/mini/drivers/alerts`。

---

## 三、非功能要求

- **身份**：与 PC 端共用一套用户体系（同一 `user_id`），不单独建账号。
- **来源标识**：所有移动端发起的单据带 `source_channel=MINI`，由后端写入。
- **轻量**：列表与首页接口字段聚合，减少请求次数；敏感配置与复杂统计不下发。
- **多端**：同一套代码支持微信小程序、钉钉小程序、H5，差异通过条件编译或运行时判断（如扫码、定位）。

---

## 四、实现顺序建议

1. **Phase 1（P0）**：登录与绑定、出土单位首页（含快捷入口骨架）。
2. **Phase 2（P1）**：出土拍照、打卡异常申报、延期申报、司机专属首页。
3. **Phase 3（P2）**：问题反馈；与审批/消息联动的状态回写与推送（若本期包含）。

---

## 五、与后端、PC 的衔接

- 后端接口按 `技术方案_小程序.md` 的 `/api/mini/*` 实现；本需求文档不规定后端表结构，仅约定接口语义与验收标准。
- 审批、消息、项目/合同数据由后端统一处理；小程序仅发起与展示状态，不做审批配置与复杂统计。
- 附件上传复用现有统一文件服务，小程序只传 file_id 或 multipart 由后端落库。

## 六、实现说明（xngl-mp）

| 功能 | 页面/路径 | 实现要点 |
|------|-----------|----------|
| 登录与绑定 | pages/login/index | 账号+密码登录，调用 POST /api/mini/auth/login；失败或接口未就绪时支持模拟登录进入首页 |
| 出土单位首页 | pages/index/index | 当前单位、可用项目列表（GET current、GET projects）、快捷入口（拍照/异常申报/延期/反馈/司机端） |
| 出土拍照 | pages/photos/index | 列表（GET /api/mini/photos）、拍照/选图后弹窗补备注与项目、提交 POST /api/mini/photos；接口失败时展示模拟数据 |
| 打卡异常申报 | pages/checkin-exceptions/index | 列表 + 新增申报弹窗（原因、类型），POST/GET /api/mini/checkin-exceptions |
| 延期申报 | pages/delay-applies/index | 列表 + 新增申报弹窗（延期至日期、原因、项目），POST/GET /api/mini/delay-applies |
| 问题反馈 | pages/feedbacks/index | 列表 + 新增反馈弹窗（内容），POST/GET /api/mini/feedbacks |
| 司机专属 | pages/driver/index | 司机工作台：任务/预警/处置证卡片、今日任务列表、待处理预警列表；数据来自 GET /api/mini/drivers/home，失败时模拟数据 |

**请求封装**：`src/utils/request.ts` 统一 baseURL、token（getToken/setToken/clearToken）、各接口方法；401 时清 token 并跳转登录。

**后端未就绪时**：列表与首页接口在 catch 中回退为模拟数据，登录在接口失败时使用模拟 token 以便本地/联调。

**文档结束**。实现时以本需求文档为验收依据，技术细节以 `技术方案_小程序.md` 与 `xngl-mp` 工程为准。
