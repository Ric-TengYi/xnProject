# 消纳场地技术方案

## 1. 模块建设目标
建立覆盖场地建档、消纳运营、设备接入、资料管理、结算管理和安全管理的场地治理体系。

## 2. 模块技术范围
消纳场地模块技术范围覆盖前端页面、后台接口、业务服务、数据模型、审批联动、消息通知、日志审计、导入导出、移动端或第三方对接等能力。

## 3. 总体能力设计
消纳场地模块负责承载现场运营与平台结算两类能力，需要同时满足场地台账、容量监测、人工消纳、地磅协同、资料归档、安全台账和运营配置等需求。技术上采用统一前后端分层架构，前端负责界面编排与交互约束，后端负责业务校验、流程编排、数据存储、异步任务和接口开放。

## 4. 功能点技术实现方案

### 4.1 场地列表

**目标能力**
建设“场地列表”的标准化服务能力，满足支持消纳场地列表展示，能直观展示场地的当前容量等数据的业务目标，并为后续审批、预警、统计、移动端和第三方联动提供统一接口支撑。

**系统提供能力说明**
系统需提供主数据维护、状态流转、日志留痕、接口开放、消息通知和统计汇总等能力，并支持按组织、项目、场地、车辆、合同等维度扩展查询范围。

**参与角色**
- 场地管理员、运营人员、财务人员、安全管理员、平台管理员

**前后端功能组成**
- 前端：管理列表、详情抽屉、表单页签、流程时间线、导入导出弹窗、地图或图表视图。
- 后端：控制器接口、服务编排、规则校验、数据落库、异步任务、日志审计、消息触发。

**业务接口说明**
- `GET /api/sites`：承载“场地列表”相关的查询、提交、处理或结果获取能力。
- `GET /api/sites/{id}`：承载“场地列表”相关的查询、提交、处理或结果获取能力。

**输入输出参数说明**
- 输入：主业务对象编号、组织范围、时间范围、状态、关键业务参数、附件或现场信息。
- 输出：业务结果状态、结果数据明细、日志编号、流程节点信息、导出或下载结果。

**业务流程说明**
1. 前端页面或移动端提交业务请求，并携带当前用户、组织范围和业务参数。
2. 接口层完成参数校验、权限校验、状态校验和幂等校验。
3. 服务层执行业务规则计算、状态流转、关联对象更新和消息触发。
4. 数据层完成主表、明细表、关联表、日志表和统计快照写入。
5. 异步任务根据需要执行导出、同步、推送、回调、报表汇总或预警生成。

**数据存储与状态流转**
- 主业务数据写入模块主表，明细、附件、流程、消息、日志分别写入关联表或扩展表。
- 状态流转需统一采用状态机控制，至少覆盖“草稿、已提交、处理中、已完成、已退回、已关闭/已作废”等状态组合中的适用状态。

**权限控制**
- 接口级权限控制：按菜单、按钮、角色和组织范围控制访问。
- 数据级权限控制：按用户所属组织、授权场地、授权项目、授权车辆或合同范围过滤结果。

**异常处理机制**
- 接口参数校验失败返回统一错误码与字段定位信息。
- 权限不足或数据越权时直接拒绝，并记录安全审计日志。
- 并发提交或重复处理时通过幂等键、状态机或唯一约束防止重复写入。

**日志与审计要求**
- 记录请求来源、操作人、操作时间、关键参数、前后状态和处理结果。
- 涉及审批、推送、同步、导入导出、删除、解禁等动作时需保留完整审计链。

**可扩展设计**
- 接口层预留版本控制、扩展字段和第三方回调适配能力。
- 数据层预留编码、来源平台、批次号、同步标识和外部对象编号等扩展字段。

**业务流程说明**
1. 前端页面或移动端提交业务请求，并携带当前用户、组织范围和业务参数。

3. 服务层执行业务规则计算、状态流转、关联对象更新和消息触发。
4. 数据层完成主表、明细表、关联表、日志表和统计快照写入。

- 权限不足或数据越权时直接拒绝，并记录安全审计日志。
- 并发提交或重复处理时通过幂等键、状态机或唯一约束防止重复写入。

**日志与审计要求**
- 记录请求来源、操作人、操作时间、关键参数、前后状态和处理结果。
- 涉及审批、推送、同步、导入导出、删除、解禁等动作时需保留完整审计链。

**可扩展设计**
- 接口层预留版本控制、扩展字段和第三方回调适配能力。
- 数据层预留编码、来源平台、批次号、同步标识和外部对象编号等扩展字段。

- 场地管理员、运营人员、财务人员、安全管理员、平台管理员

**前后端功能组成**
- 前端：管理列表、详情抽屉、表单页签、流程时间线、导入导出弹窗、地图或图表视图。
- 后端：控制器接口、服务编排、规则校验、数据落库、异步任务、日志审计、消息触发。

**业务接口说明**
- `GET /api/sites`：承载“运营配置”相关的查询、提交、处理或结果获取能力。
- `POST /api/sites`：承载“运营配置”相关的查询、提交、处理或结果获取能力。
- `PUT /api/sites/{id}`：承载“运营配置”相关的查询、提交、处理或结果获取能力。

**输入输出参数说明**
- 输入：主业务对象编号、组织范围、时间范围、状态、关键业务参数、附件或现场信息。
- 输出：业务结果状态、结果数据明细、日志编号、流程节点信息、导出或下载结果。

**业务流程说明**
1. 前端页面或移动端提交业务请求，并携带当前用户、组织范围和业务参数。
2. 接口层完成参数校验、权限校验、状态校验和幂等校验。
3. 服务层执行业务规则计算、状态流转、关联对象更新和消息触发。
4. 数据层完成主表、明细表、关联表、日志表和统计快照写入。
5. 异步任务根据需要执行导出、同步、推送、回调、报表汇总或预警生成。

**数据存储与状态流转**
- 主业务数据写入模块主表，明细、附件、流程、消息、日志分别写入关联表或扩展表。
- 状态流转需统一采用状态机控制，至少覆盖“草稿、已提交、处理中、已完成、已退回、已关闭/已作废”等状态组合中的适用状态。

**权限控制**
- 接口级权限控制：按菜单、按钮、角色和组织范围控制访问。
- 数据级权限控制：按用户所属组织、授权场地、授权项目、授权车辆或合同范围过滤结果。

**异常处理机制**
- 接口参数校验失败返回统一错误码与字段定位信息。
- 权限不足或数据越权时直接拒绝，并记录安全审计日志。
- 并发提交或重复处理时通过幂等键、状态机或唯一约束防止重复写入。

**日志与审计要求**
- 记录请求来源、操作人、操作时间、关键参数、前后状态和处理结果。
- 涉及审批、推送、同步、导入导出、删除、解禁等动作时需保留完整审计链。

**可扩展设计**
- 接口层预留版本控制、扩展字段和第三方回调适配能力。
- 数据层预留编码、来源平台、批次号、同步标识和外部对象编号等扩展字段。

## 5. 核心数据对象设计
- 场地主档 Site
- 场地资料 SiteDocument
- 场地设备 SiteDevice
- 场地运营配置 SiteOperationConfig
- 场地人员配置 SitePersonnelConfig
- 场地测绘记录 SiteSurveyRecord
- 场地结算单 SiteSettlement

## 6. 业务接口设计
- 统一接口前缀建议采用 `/api/sites` 或对应移动端、专项前缀，实现前后端接口分层清晰。
- 查询类接口支持分页、排序、条件过滤、字典翻译和状态聚合。
- 变更类接口支持幂等控制、状态校验、附件提交、审批提交流程和消息触发。

## 7. 业务流程设计
- 业务流程统一遵循“发起/采集 -> 校验 -> 提交/处理 -> 审批/同步 -> 结果回写 -> 留痕/通知”的流程骨架。
- 涉及统计、导出、同步、报表和预警生成的场景，采用异步任务和批处理机制降低接口阻塞风险。

## 8. 权限与数据范围控制
- 统一接入组织、用户、角色和数据范围控制体系，按单位、项目、场地、车辆、合同等维度裁剪数据可见范围。
- 移动端、第三方接口和后台接口均需绑定独立的身份凭据和访问授权。

## 9. 异常处理与补偿机制
- 关键接口统一返回错误码、错误信息、追踪编号和建议处理方式。
- 异步任务支持失败重试、批次追踪、人工补录、对账补偿和回滚控制。

## 10. 日志审计与追溯要求
- 记录接口访问日志、业务操作日志、审批日志、同步日志、错误日志和消息推送回执。
- 关键业务对象支持按单号、主键、批次号、外部编码追溯全链路处理过程。

## 11. 与第三方或其他模块协同方式
- 与组织权限体系协同实现统一身份和数据授权。
- 与审批配置、消息中心、日志中心协同实现流程驱动、消息触达和可审计治理。
- 与平台对接、小程序、统计分析、数据看板协同实现跨平台数据共享和能力复用。

## 12. 性能与扩展性要求
- 列表和报表需支持分页、条件索引、聚合优化和异步导出。
- 地图、轨迹、视频、地磅、同步日志等大体量数据建议分表、归档或采用冷热分层策略。

## 13. 本模块交付说明
消纳场地模块应作为独立可交付能力建设，既满足甲方当前招标要求，也为后续扩展第三方对接、移动端深化、规则引擎和经营分析能力保留扩展接口。
