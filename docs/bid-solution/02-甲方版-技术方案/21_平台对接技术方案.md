# 平台对接技术方案

## 1. 模块建设目标
建立覆盖政务网、定位/GPS、视频、地磅、统一身份认证与单点登录的统一对接能力，实现对接配置、数据同步、日志追踪、异常补偿和运维监控标准化。

## 2. 模块技术范围
平台对接模块技术范围包括：
- 对接配置管理
- 政务网同步服务
- 定位/GPS 轨迹接入服务
- 视频设备与通道接入服务
- 地磅数据接入与对账服务
- 单点登录票据交换服务
- 异步任务、同步日志、失败补偿和运维监控服务

## 3. 总体能力设计
平台对接模块采用“接入配置 + 适配器服务 + 同步任务 + 原始日志 + 业务回写 + 对账补偿 + 运维监控”的标准架构：
- 接入配置层负责维护地址、协议、鉴权和映射参数。
- 适配器层负责协议转换、签名校验、连接复用和异常分类。
- 任务层负责定时同步、手动同步、重试、补传和对账。
- 数据层负责原始报文、中间记录、业务映射结果和同步日志留存。
- 服务层向业务模块提供统一查询、回写、日志和异常处理能力。

## 4. 功能点技术实现方案

### 4.1 政务网数据对接
**目标能力**
接入政务平台准运证、处置证等监管数据，支持定时同步、人工触发、批次追踪、失败补偿和业务回写。

**系统提供能力说明**
系统需提供连接配置、同步任务、同步日志、原始报文留存、字段映射校验、业务落库和失败重试能力。

**参与角色**
- 集成管理员
- 第三方实施人员
- 运维管理员

**前后端功能组成**
- 前端：政务对接配置页、同步日志页、失败明细页、批次详情页。
- 后端：政务同步控制器、政务适配器、同步任务服务、日志服务、补偿服务。

**业务接口说明**
- `GET /api/platform-integrations/gov/configs`：查询政务网连接配置。
- `POST /api/platform-integrations/gov/configs`：新增或更新政务网连接配置。
- `POST /api/platform-integrations/gov/test-connection`：执行政务网连通性和鉴权测试。
- `POST /api/platform-integrations/gov/sync`：发起一次政务网同步任务。
- `GET /api/platform-integrations/gov/tasks/{taskId}`：查询指定同步任务状态和统计结果。
- `GET /api/platform-integrations/gov/logs`：按批次、对象、时间范围查询同步日志。
- `POST /api/platform-integrations/gov/logs/{logId}/replay`：对失败记录执行重放或补偿。

**输入输出参数说明**
- 输入：配置编号、同步范围、批次号、时间窗、对象类型、失败记录编号。
- 输出：任务编号、同步结果统计、失败明细、业务回写结果、异常码和日志编号。

**业务流程说明**
1. 前端提交同步或配置请求。
2. 接口层完成参数、鉴权、权限和幂等校验。
3. 适配器层向政务网发起拉取请求并记录原始请求/响应报文。
4. 解析层把监管数据转换为平台中间对象，执行字段校验和重复校验。
5. 服务层完成业务映射和业务主表回写。
6. 失败记录进入补偿队列并支持人工重放。

**数据存储与状态流转**
- 主表：`platform_integration_config`、`platform_sync_task`、`platform_sync_log`。
- 中间对象：政务原始报文、解析结果、校验结果、业务回写结果。
- 任务状态：`CREATED`、`RUNNING`、`PARTIAL_SUCCESS`、`SUCCESS`、`FAILED`。

**权限控制**
- 仅集成管理员和运维管理员可维护连接配置。
- 日志查询按组织与对接域授权，不允许业务用户查看敏感报文。

**异常处理机制**
- 连接失败：记录地址、响应超时和错误码，支持自动重试。
- 鉴权失败：中止任务并冻结当前配置为“待修复”。
- 数据解析失败：记录失败字段、原始对象编码和失败明细。
- 回写失败：保留中间结果，支持逐条重放。

**日志与审计要求**
- 记录请求报文摘要、响应报文摘要、批次号、对象编号、处理耗时和重试次数。
- 关键配置变更记录操作人、时间、前后配置差异。

**可扩展设计**
- 适配器层支持扩展更多监管平台协议和对象类型。
- 中间对象保留来源平台、批次号、外部对象编号等扩展字段。

### 4.2 定位/GPS 对接
**目标能力**
接入车辆实时位置、历史轨迹、围栏事件、停留点和异常补传数据，为预警、地图、小程序和监管业务提供标准轨迹服务。

**系统提供能力说明**
系统需提供轨迹接入、轨迹查询、围栏判定、异常补传、轨迹压缩和轨迹共享服务。

**参与角色**
- 集成管理员
- 运维管理员
- 监管平台管理员

**前后端功能组成**
- 前端：定位平台配置页、轨迹接入监控页、围栏事件日志页。
- 后端：定位接收控制器、轨迹服务、围栏判定服务、补传任务服务。

**业务接口说明**
- `GET /api/platform-integrations/location/configs`：查询定位平台配置。
- `POST /api/platform-integrations/location/configs`：保存定位平台配置。
- `POST /api/platform-integrations/location/test-connection`：测试定位平台连通性。
- `POST /api/platform-integrations/location/push`：接收第三方推送的轨迹点。
- `GET /api/platform-integrations/location/vehicles/{vehicleId}/latest`：查询车辆最新位置。
- `GET /api/platform-integrations/location/vehicles/{vehicleId}/tracks`：按时间范围查询历史轨迹。
- `GET /api/platform-integrations/location/fence-events`：查询围栏进出事件。
- `POST /api/platform-integrations/location/tracks/retransmit`：发起轨迹补传任务。

**输入输出参数说明**
- 输入：车辆编码、设备编码、轨迹时间、经纬度、速度、方向、围栏编号、补传时间窗。
- 输出：轨迹接收结果、围栏事件、停留点、补传结果、异常轨迹日志。

**业务流程说明**
1. 定位平台以推送或拉取方式传输轨迹点。
2. 接收层校验时间、坐标、车辆映射和设备状态。
3. 合格轨迹写入轨迹分片表并触发围栏判定。
4. 围栏事件与异常轨迹分别写入事件表和异常表。
5. 轨迹服务向预警、地图、小程序和信息查询模块提供查询服务。
6. 对中断区间发起补传并回写补传结果。

**数据存储与状态流转**
- 主表：`location_config`、`vehicle_track_point`、`alert_fence_event`、`track_retransmit_task`。
- 状态：`RECEIVED`、`VALID`、`ABNORMAL`、`RETRANSMITTED`、`CLOSED`。

**权限控制**
- 轨迹查询按车辆、组织、项目、场地授权范围过滤。
- 围栏配置和补传任务仅限管理员角色操作。

**异常处理机制**
- 坐标异常：直接丢弃并记录异常轨迹日志。
- 轨迹中断：生成补传建议并提示最后有效时间。
- 设备离线：写入离线告警并通知运维。

**日志与审计要求**
- 记录轨迹接收量、异常量、围栏命中量、补传次数和处理耗时。
- 围栏策略和补传操作需记录操作人和生效范围。

**可扩展设计**
- 支持多定位供应商适配。
- 支持轨迹压缩、停留点分析和电子围栏策略扩展。

### 4.3 视频对接
**目标能力**
接入项目/场地视频设备、抓拍机、视频通道和播放能力，支持实时查看、抓拍留存、通道监测和异常告警。

**系统提供能力说明**
系统需提供设备映射、通道管理、播放令牌生成、抓拍记录、心跳检测和告警能力。

**参与角色**
- 集成管理员
- 运维管理员
- 场地管理员

**前后端功能组成**
- 前端：设备管理页、通道管理页、实时查看页、抓拍记录页。
- 后端：视频设备控制器、通道服务、播放凭证服务、设备心跳服务。

**业务接口说明**
- `GET /api/platform-integrations/video/devices`：查询视频设备列表。
- `POST /api/platform-integrations/video/devices`：新增或更新视频设备映射。
- `GET /api/platform-integrations/video/devices/{deviceId}/channels`：查询设备通道列表。
- `POST /api/platform-integrations/video/devices/{deviceId}/test`：测试指定设备连通性。
- `GET /api/platform-integrations/video/channels/{channelId}/play-token`：获取播放令牌。
- `POST /api/platform-integrations/video/channels/{channelId}/capture`：发起截图抓拍。
- `GET /api/platform-integrations/video/logs`：查询设备接入与播放日志。

**输入输出参数说明**
- 输入：设备编码、通道编码、所属项目/场地、播放协议、抓拍参数。
- 输出：设备状态、通道状态、播放令牌、抓拍结果、异常日志。

**业务流程说明**
1. 管理员维护设备和通道映射。
2. 系统定时拉取设备状态并记录心跳。
3. 用户查看视频时，平台生成短期播放令牌。
4. 前端使用播放令牌拉取视频流并完成展示。
5. 异常通道生成告警并支持重试或抓拍降级。

**数据存储与状态流转**
- 主表：`video_device`、`video_channel`、`video_capture_record`、`video_access_log`。
- 状态：`ONLINE`、`OFFLINE`、`AUTH_FAILED`、`DISABLED`。

**权限控制**
- 视频查看按项目、场地、设备权限控制。
- 抓拍和配置操作需单独按钮权限。

**异常处理机制**
- 通道不可用：返回明确故障类型并触发告警。
- 播放令牌失效：要求重新获取令牌。
- 设备离线：限制实时查看并保留离线时间。

**日志与审计要求**
- 记录播放请求、抓拍请求、设备状态变化和鉴权失败信息。
- 保存抓拍记录与操作留痕。

**可扩展设计**
- 支持更多视频平台协议和抓拍能力扩展。
- 支持 PTZ 控制、录像回放和 AI 抓拍扩展。

### 4.4 地磅数据对接
**目标能力**
接入地磅称重记录和本地控制指令，完成过磅采集、异常识别、人工对账和业务回写。

**系统提供能力说明**
系统需提供设备配置、记录接收、业务匹配、异常对账、控制指令、回写服务。

**参与角色**
- 集成管理员
- 场地管理员
- 运维管理员

**前后端功能组成**
- 前端：地磅设备页、称重记录页、异常对账页、控制指令页。
- 后端：地磅接入控制器、记录服务、对账服务、控制指令服务。

**业务接口说明**
- `GET /api/platform-integrations/weighbridge/devices`：查询地磅设备列表。
- `POST /api/platform-integrations/weighbridge/devices`：新增或更新地磅设备配置。
- `POST /api/platform-integrations/weighbridge/push`：接收地磅推送的称重记录。
- `GET /api/platform-integrations/weighbridge/records`：按设备、车牌、时间查询称重记录。
- `POST /api/platform-integrations/weighbridge/reconcile`：对异常记录执行人工对账。
- `POST /api/platform-integrations/weighbridge/commands`：下发控制指令。
- `GET /api/platform-integrations/weighbridge/logs`：查询地磅接入和控制日志。

**输入输出参数说明**
- 输入：设备编码、车牌、称重时间、毛重、皮重、净重、对账动作、控制指令。
- 输出：记录接收结果、对账结果、回写结果、指令执行结果、异常码。

**业务流程说明**
1. 设备推送称重记录或本地控制软件上送记录。
2. 接口层校验设备、记录号、重量和车牌信息。
3. 服务层完成业务匹配和异常识别。
4. 正常记录回写消纳清单或结算台账。
5. 异常记录进入对账池并支持人工处理。
6. 控制指令执行结果写入指令日志。

**数据存储与状态流转**
- 主表：`weighbridge_device`、`weighbridge_record`、`weighbridge_reconcile_log`、`weighbridge_command_log`。
- 状态：`RECEIVED`、`MATCHED`、`ABNORMAL`、`CONFIRMED`、`IGNORED`。

**权限控制**
- 对账和控制指令仅对场地管理员与运维管理员开放。
- 称重记录查询按场地和组织授权范围过滤。

**异常处理机制**
- 重量异常：标记异常并阻止自动回写。
- 车牌不匹配：进入人工确认池。
- 指令执行失败：记录设备返回值并支持重发。

**日志与审计要求**
- 记录称重原始报文、对账动作、回写结果和控制指令回执。
- 关键记录支持按设备、车牌、批次、场地追溯。

**可扩展设计**
- 支持借磅场景、测绘结算联动和多品牌设备适配。
- 支持图片抓拍、票据打印和排队控制扩展。

### 4.5 统一身份认证与单点登录
**目标能力**
实现跨平台票据创建、票据交换、会话恢复、统一退出和登录审计，满足跨平台免密访问需求。

**系统提供能力说明**
系统需提供身份源配置、票据创建、票据校验、会话映射、统一退出和失败重试能力。

**参与角色**
- 集成管理员
- 系统管理员
- 运维管理员

**前后端功能组成**
- 前端：SSO 配置页、票据日志页、失败交换页。
- 后端：SSO 控制器、票据服务、会话服务、映射服务、退出服务。

**业务接口说明**
- `GET /api/platform-integrations/sso/configs`：查询 SSO 配置。
- `POST /api/platform-integrations/sso/configs`：保存 SSO 配置。
- `POST /api/platform-integrations/sso/test-connection`：测试身份源连通性与票据协议。
- `POST /api/platform-integrations/sso/tickets`：创建登录票据。
- `POST /api/platform-integrations/sso/exchange`：执行票据交换并建立本地会话。
- `POST /api/platform-integrations/sso/logout`：执行统一退出。
- `GET /api/platform-integrations/sso/logs`：查询票据和会话日志。

**输入输出参数说明**
- 输入：身份源编号、票据编号、签名信息、回调地址、用户映射信息。
- 输出：票据状态、交换结果、会话标识、失败原因、日志编号。

**业务流程说明**
1. 外部系统创建票据或跳转票据。
2. 平台校验票据签名、时效和回调来源。
3. 根据身份映射恢复本地用户与会话。
4. 登录成功后写入登录日志和票据日志。
5. 退出时同时销毁本地会话和跨平台票据状态。

**数据存储与状态流转**
- 主表：`sso_config`、`sso_ticket`、`sso_exchange_log`。
- 状态：`CREATED`、`EXCHANGED`、`EXPIRED`、`FAILED`、`LOGOUT`。

**权限控制**
- SSO 配置仅系统管理员和集成管理员可维护。
- 票据日志需受限访问，不向普通业务用户开放。

**异常处理机制**
- 票据过期：要求重新发起登录。
- 签名失败：中止交换并记录安全事件。
- 身份未映射：阻止登录并提示绑定本地账号。

**日志与审计要求**
- 记录票据创建、交换、退出、失败原因、来源 IP、用户标识和会话标识。
- 所有跨平台登录行为纳入统一登录审计体系。

**可扩展设计**
- 支持多身份源和多协议接入。
- 支持移动端单点登录和统一退出扩展。

## 5. 核心数据对象设计
- 对接配置 `platform_integration_config`
- 同步任务 `platform_sync_task`
- 同步日志 `platform_sync_log`
- 轨迹接收记录 `vehicle_track_point`
- 围栏事件 `alert_fence_event`
- 视频设备 `video_device`
- 视频通道 `video_channel`
- 地磅记录 `weighbridge_record`
- SSO 票据 `sso_ticket`

## 6. 业务接口设计
- 统一使用 `/api/platform-integrations/{domain}/...` 作为对接域前缀，避免不同对接能力混用接口。
- 对接域建议拆分为 `gov`、`location`、`video`、`weighbridge`、`sso` 五类。
- 所有接口统一返回 `traceId`、`taskId`、`status`、`message`、`errorCode` 等字段，便于追踪。

## 7. 业务流程设计
- 配置类流程：维护配置 -> 测试连通 -> 生效配置 -> 审计留痕。
- 同步类流程：发起任务 -> 接收数据 -> 校验映射 -> 写入中间表 -> 回写业务表 -> 记录日志 -> 失败补偿。
- 设备类流程：设备接入 -> 心跳监测 -> 业务消费 -> 异常告警 -> 运维处理。
- 单点登录流程：创建票据 -> 票据交换 -> 建立会话 -> 访问业务 -> 统一退出。

## 8. 权限与数据范围控制
- 配置、测试、重试、对账、控制指令等动作需独立按钮权限。
- 日志查询需按对接域、组织和敏感级别裁剪。
- 跨模块共享的轨迹、视频、地磅、政务数据在被业务模块消费时仍需执行本地数据范围控制。

## 9. 异常处理与补偿机制
- 建立统一异常分类：连接异常、鉴权异常、协议异常、数据异常、业务映射异常、回写异常、超时异常。
- 对同步、补传、对账、播放、票据交换等任务提供自动重试和人工补偿入口。
- 对批量任务支持批次级回溯与逐条补偿，避免整批重跑。

## 10. 日志审计与追溯要求
- 所有请求保留请求时间、来源、对象编号、批次号、任务号、响应结果、耗时和重试次数。
- 关键配置变更、票据交换、控制指令和人工对账必须留痕。
- 对接日志需支持按项目、场地、设备、车辆、批次、外部编码检索。

## 11. 与第三方或其他模块协同方式
- 政务网数据对接与处置证、项目管理、合同结算协同。
- 定位/GPS 对接与系统预警、数据看板、信息查询、小程序协同。
- 视频和地磅与消纳场地、项目管理、事件管理协同。
- SSO 与组织、人员、角色、消息、小程序协同。

## 12. 性能与扩展性要求
- 轨迹、地磅、日志等高频数据需采用分片、归档和异步处理策略。
- 视频播放令牌和轨迹查询需支持缓存与短期会话控制。
- 对接域适配器应支持多供应商扩展和协议扩展。

## 13. 本模块交付说明
平台对接模块应作为甲方正式方案中的核心技术能力模块交付，并与《第三方系统对接专项方案》形成“模块落地 + 总体规范”的双层说明关系。
