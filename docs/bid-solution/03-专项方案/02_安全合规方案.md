# 安全合规方案

## 1. 安全建设目标
平台安全建设以“身份可信、访问可控、数据可管、行为可审、风险可追”为目标，满足甲方对政务类平台、运营管理平台和移动端场景的安全要求。

## 2. 等保二级配合方案
- 按照等保二级要求梳理安全管理制度、资产台账、账号权限、日志留痕和应急预案。
- 配合完成网络边界、主机安全、应用安全、数据安全和安全管理中心相关整改。
- 输出配套的安全制度、权限台账、日志审计方案和备份恢复方案，支撑测评取证。

## 3. 信创适配要求
- 支持国产服务器、国产操作系统、国产数据库和国产中间件部署适配。
- 应用层避免强耦合特定商业软件组件，接口和驱动保留替换能力。
- 对浏览器端、移动端和打印导出能力优先采用标准化协议和中立格式。

## 4. 数据安全方案
- 核心业务数据按组织、项目、场地、车辆、合同等维度实施分级分类管理。
- 敏感信息如手机号、证照号、定位信息、外部凭据采用脱敏展示和加密存储。
- 数据导出、批量查询、外部同步需记录操作者、范围、时间、结果和用途。

## 5. 身份认证与权限控制
- 后台统一采用账号密码 + 短信/二次校验等认证方式。
- 小程序支持账号密码、短信验证码、微信绑定与 openId 免登等多因子能力。
- 权限体系采用“组织 + 用户 + 角色 + 菜单 + 按钮 + 数据范围”组合控制模型。

## 6. 操作审计与日志留痕
- 所有登录、登出、查询、导出、审批、同步、删除、解禁、参数修改等关键动作必须落审计日志。
- 错误日志、同步日志、审批日志和消息日志需保留追踪编号，支持跨模块追溯。

## 7. 接口安全与传输安全
- 外部接口统一使用 HTTPS、签名、时间戳、Token、白名单等方式保证传输安全。
- 防止重放攻击、越权访问、非法参数注入和批量刷接口行为。
- 对文件上传、图片上传和导入文件进行类型校验、病毒扫描和内容安全检查。

## 8. 数据备份与恢复要求
- 业务库、日志库、附件库和配置库分别制定备份策略。
- 关键数据支持每日增量备份、周期性全量备份和异地备份。
- 恢复能力需覆盖单表恢复、库级恢复和灾难恢复演练。
