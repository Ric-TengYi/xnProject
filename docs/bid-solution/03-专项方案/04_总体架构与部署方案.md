# 总体架构与部署方案

## 1. 平台总体建设思路
平台采用“统一主数据、统一流程、统一消息、统一集成、统一审计、统一移动入口”的建设思路，围绕渣土运输与消纳监管业务构建可扩展、可集成、可审计的平台体系。

## 2. 总体业务架构
业务架构分为五层：
1. 基础管理层：组织、人员、角色、字典、参数、审批配置、日志。
2. 经营业务层：单位、项目、合同、处置证、场地、车辆、车队、结算。
3. 监管风控层：预警、违规车辆、研判模型、安全管理、事件管理。
4. 展示服务层：统计分析、数据看板、信息查询、消息中心、小程序。
5. 集成支撑层：政务网、定位、视频、地磅、SSO、短信、推送。

## 3. 应用架构设计
- 前端采用管理后台 + 小程序双端架构。
- 后端采用分层服务架构，划分控制器层、业务服务层、数据访问层和集成适配层。
- 异步任务统一承载导出、同步、预警、消息和统计快照生成。

## 4. 数据架构设计
- 主数据层：组织、用户、单位、项目、场地、车辆、合同、处置证。
- 业务数据层：打卡、消纳、地磅、审批、事件、预警、维修、维保、报表。
- 审计数据层：登录日志、操作日志、错误日志、同步日志、消息日志。
- 扩展数据层：视频通道、定位轨迹、移动端记录、学习记录、电子围栏。

## 5. 集成架构设计
- 采用“接入配置 + 接口适配 + 同步任务 + 同步日志 + 对账补偿”的标准集成模式。
- 对政务网、地磅、定位、视频、短信、推送、SSO 提供统一适配框架。

## 6. 部署架构设计
- 前端静态资源通过 Web 服务器或网关发布。
- 后端服务、任务服务、集成服务可按业务规模拆分部署。
- 数据库、缓存、文件存储和日志服务独立部署并做环境隔离。

## 7. 高可用与扩展性设计
- 核心能力按集群部署，支持弹性扩容。
- 高并发读多写少场景可引入缓存，历史轨迹和日志数据采用分区或归档策略。
- 新增业务模块或第三方对接时，通过字典、参数、审批、消息、日志和接口规范实现快速接入。

## 8. 安全架构设计
- 统一接入认证授权、接口签名、日志审计、数据脱敏、备份恢复和运维监控能力。
- 移动端、第三方接口、后台管理、文件存储和消息推送均采用分层安全控制策略。
