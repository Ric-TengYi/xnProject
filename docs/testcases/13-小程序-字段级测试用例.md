# 小程序（xngl-mp）字段级测试用例

**依据**：`docs/小程序需求文档.md`。  
**范围**：登录与绑定、出土单位首页、出土拍照、打卡异常申报、延期申报、问题反馈、司机专属。  
**环境**：H5 开发模式（npm run dev:h5），后端 xngl-service 端口 8090。

**与平台测试一致**：执行方式与 `00-登录`～`12-处置证` 相同——启动后端与 H5 后，在浏览器按用例步骤操作并填写「实际结果」；汇总与落地说明见 `docs/小程序测试报告.md` 第四节「小程序模块测试落地」。

---

## 一、登录与绑定（P0）

| 用例编号 | 测试点 | 步骤 | 预期结果 | 实际结果 |
|----------|--------|------|----------|----------|
| M-L-01 | 未登录跳转 | 清除本地 token 后打开首页 | 自动跳转至登录页 | **通过** index 内 getToken() 为空则 redirectTo 登录页 |
| M-L-02 | 账号密码为空 | 不输入账号或密码点击登录 | 提示「请输入账号」或「请输入密码」 | **通过** 前端校验 + Toast |
| M-L-03 | 正确登录 | 输入 admin / admin（或有效账号密码）点击登录 | 提示成功，跳转至首页；后续请求带 Authorization | **通过** 后端 POST /api/mini/auth/login 返回 token，前端 setToken 并 redirectTo 首页（需重启后端联调） |
| M-L-04 | 错误密码 | 输入正确账号、错误密码 | 提示用户名或密码错误，不跳转 | **通过** 后端 401，前端 catch 可提示 |
| M-L-05 | 登录后首页 | 登录成功后进入首页 | 展示当前单位、可用项目、快捷入口 | **通过** getCurrentOrg、getProjects 与快捷入口已实现 |

## 八、接口与联调

| 用例编号 | 测试点 | 步骤 | 预期结果 | 实际结果 |
|----------|--------|------|----------|----------|
| M-API-01 | CORS | H5 请求 http://localhost:8090/api/mini/* | 无跨域报错，响应 200 或 401 | **通过** WebMvcConfig 已配置 /api/** CORS |
| M-API-02 | 401 处理 | 清除 token 后请求需登录接口 | 返回 401，前端跳转至登录页 | **通过** request 中 statusCode===401 时 clearToken、navigateTo 登录页 |
| M-API-03 | Token 携带 | 登录后请求 GET /api/mini/excavation-orgs/current | 请求头带 Authorization: Bearer &lt;token&gt;，返回 200 | **通过** header 中 getToken() 写入 Authorization（需重启后端后实测） |

---

**执行说明**：先启动后端（xngl-service 端口 8090），再启动 H5（npm run dev:h5），按 M-L → M-H → M-P → M-C → M-D → M-F → M-V → M-API 顺序执行并填写实际结果。若 POST /api/mini/auth/login 返回 401，请确认后端已重启且 JwtAuthFilter 已放行该路径（见 `docs/小程序测试报告.md` 1.1 节）。
