# 登录模块 - 字段级测试用例

## 一、账号登录

| 用例编号 | 测试点 | 字段/限制 | 步骤 | 预期结果 | 实际结果 |
|----------|--------|-----------|------|----------|----------|
| L-01 | 页面加载 | - | 访问 /login | 展示账号登录 Tab、验证码登录 Tab、账号框、密码框、登录按钮 | **通过** 页面含请输入账号、请输入密码、登录按钮及「验证码登录」Tab |
| L-02 | 账号必填 | required | 不填账号点登录 | 提示「请输入账号」，不请求接口 | **通过** 点击登录后输入框 invalid，校验生效 |
| L-03 | 密码必填 | required | 填账号不填密码点登录 | 提示「请输入密码」，不请求接口 | **通过** 仅填账号点登录仍留登录页，密码校验生效 |
| L-04 | 正向登录 | - | 账号 admin、密码 admin（tenantId=1）点登录 | 请求 POST /api/auth/login 返回 200，含 token、user；跳转至 / | **通过** 接口 200，返回 token、user（userId, tenantId, username, name, userType） |
| L-05 | 错误密码 | - | 账号正确、密码错误点登录 | 接口返回 401，提示「用户名或密码错误」 | **通过** 接口返回 401，message「用户名或密码错误」 |
| L-06 | 错误租户 | - | tenantId 不存在或无权限 | 接口返回 401 或 403 | 未测（需构造非法 tenantId） |

## 二、验证码登录 Tab

| 用例编号 | 测试点 | 字段/限制 | 步骤 | 预期结果 | 实际结果 |
|----------|--------|-----------|------|----------|----------|
| L-07 | 切换 Tab | - | 点击「验证码登录」 | 展示手机号输入框（无密码框） | **通过** 页面有验证码登录 Tab，切换后为手机号登录（未逐条测） |
| L-08 | 手机号必填 | required | 不填点登录 | 提示「请输入手机号」 | 未测 |
| L-09 | 手机号格式 | pattern 1[3-9]\\d{9} | 输入 123 或 12345678901 点登录 | 提示「请输入正确手机号」 | 未测 |
| L-10 | 手机号合法 | - | 输入 13800138000（后端未实现验证码时可能仍失败） | 前端校验通过，接口行为按实现 | 未测 |

## 三、接口契约

| 用例编号 | 测试点 | 步骤 | 预期结果 | 实际结果 |
|----------|--------|------|----------|----------|
| L-11 | 登录响应结构 | POST /api/auth/login 成功 | code=200, data.token, data.user (userId, tenantId, username, name, userType) | **通过** 结构符合 |
| L-12 | 未登录访问鉴权接口 | 无 Authorization 请求 /api/me | 401 或 403 | **通过** 无 token 时返回 401 |
