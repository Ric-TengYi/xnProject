# 多租户SaaS系统测试设计文档

## 一、系统权限体系推断

### 用户角色层级
1. **超级管理员** (Platform Admin)
   - 租户: 平台租户
   - 组织: 平台组织
   - 权限: 全系统管理权限

2. **租户管理员** (Tenant Admin)
   - 租户: 特定租户
   - 组织: 租户根组织
   - 权限: 租户内全部权限

3. **子组织管理员** (Org Admin)
   - 租户: 特定租户
   - 组织: 子组织
   - 权限: 自己组织及下级组织的管理权限

4. **普通用户** (User)
   - 租户: 特定租户
   - 组织: 特定组织
   - 权限: 根据角色分配的权限

---

## 二、组织隔离规则推断

### 角色隔离
- **超级管理员**: 可见所有租户的所有角色
- **租户管理员**: 只能看到自己租户的角色
- **子组织管理员**: 只能看到自己组织及下级组织的角色
- **普通用户**: 无法访问角色管理

### 用户隔离
- **超级管理员**: 可见所有租户的所有用户
- **租户管理员**: 只能看到自己租户的用户
- **子组织管理员**: 只能看到自己组织及下级组织的用户
- **普通用户**: 只能看到自己的信息

### 菜单权限隔离
- 菜单权限绑定到角色
- 用户通过角色获得菜单权限
- 不同组织的用户不应该看到其他组织的菜单

---

## 三、首页权限处理规则推断

### 登录后菜单加载逻辑
1. 后端返回用户的菜单列表
2. 如果菜单列表为空或不包含首页:
   - 前端应该重定向到第一个有权限的菜单
   - 如果没有任何菜单权限，显示无权限页面

### 实现位置
- 后端: MeController 返回菜单列表时，应该过滤出用户有权限的菜单
- 前端: MainLayout 在菜单加载后，检查是否有首页权限，如果没有则重定向

---

## 四、用户个人中心功能推断

### 左下角用户菜单应包含
1. **用户管理** (仅租户管理员及以上)
   - 查看租户内所有用户
   - 创建/编辑/删除用户
   - 重置用户密码

2. **修改密码** (所有用户)
   - 修改自己的密码

3. **个人信息** (所有用户)
   - 查看/修改自己的信息

4. **退出登录** (所有用户)
   - 清除登录状态

### 权限控制规则
- 用户管理: 仅租户管理员及以上可见
- 修改密码: 所有用户可见
- 个人信息: 所有用户可见
- 退出登录: 所有用户可见

---

## 五、角色列表分页规则推断

### 后端分页
- 后端应该返回分��数据 (pageNo, pageSize, total)
- 默认每页显示 10-20 条数据

### 前端处理
- 使用 Ant Design Pagination 组件
- 支持用户选择每页显示数量
- 加载更多或翻页时重新请求数据

- [ ] 无法访问角色管理

- [ ] 左下角不显示用户管理，显示修改密码、个人信息、退出

- [ ] 角色列表显示分页控件
- [ ] 默认显示第一页数据

- [ ] 分页信息正确显示 (当前页/总页数)

- [ ] 用户列表按组织正确过滤

#### 场景8: 菜单隔离验证
- [ ] 不同组织的用户看到的菜单不同
- [ ] 菜单权限正确绑定到角色

## 八、测试执行顺序

1. **权限体系验证** (基础)
   - 验证各角色能访问的菜单
   - 验证各角色的数据隔离

2. **功能隔离验证** (核心)
   - 验证角色管理隔离
   - 验证用户管理隔离
   - 验证菜单权限隔离

3. **UI验证** (细节)
   - 验证分页显示
   - 验证列表边界
   - 验证个人中心菜单

4. **边界情况验证** (异常)
   - 无权限用户访问受限功能
   - 无首页权限用户登录
   - 组织管理员权限边界
