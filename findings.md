## 严重问题

### 🚨 BUG-001: 多租户数据隔离完全失效 (严重)

**严重级别**: P0 - Critical

**问题描述**:
系统缺少租户隔离机制，任一租户用户可以访问、修改、删除其他租户的数据。

**复现步骤**:
1. 创建租户2用户 `tenant2_user`
2. 使用租户2用户登录
3. 该用户可以：
   - 查询租户1的组织树
   - 查询租户1的用户详情
   - 修改租户1的用户信息
   - 删除租户1的用户
   - 在租户1创建组织

**根本原因**:
`MybatisPlusConfig.java` 只配置了分页拦截器，缺少 `TenantLineInnerInterceptor` 租户拦截器：

```java
// 当前配置 - 缺少租户拦截器
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
  MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
  interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
  return interceptor;
}
```

**安全影响**:
- 数据泄露风险：租户A可查看租户B的所有数据
- 数据篡改风险：租户A可修改租户B的数据
- 数据丢失风险：租户A可删除租户B的数据

**修复建议**:
1. 添加 MyBatis-Plus 租户拦截器
2. 实现租户上下文（从JWT获取当前用户租户ID）
3. 所有实体操作自动添加 tenant_id 条件
